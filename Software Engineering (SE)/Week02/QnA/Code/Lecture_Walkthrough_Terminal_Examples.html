Lecture Walkthrough: Version Control

1. Terminal Refresher

Activity: Create a project directory and navigate to it using the following commands:

  • mkdir project - Create a directory named project.
  • cd project - Navigate into the project directory.
  • touch index.html - Create a file named index.html.
  • ls - List the files and directories in the current directory.
  • pwd - Display the current working directory.
  • cat index.html - Display the contents of index.html.
  • rm index.html - Remove the index.html file.

Ask learners what they think they can't do on the terminal compared to a GUI.

Engaging Question: What do you find most challenging about using the terminal compared to a GUI?

On-stage Task: Invite a learner to navigate directories and demonstrate basic terminal commands.

2. Git Basics

Local Git Installation: Ensure Git is installed and configure user settings using git config --global.

Workflow Demo:

  • Initialize a repository: git init.
  • Create a file (README.md) and add content.
  • Stage and commit changes: git add and git commit -m.

Engaging Questions:

  • Why do you think Git requires staging changes before committing them?
  • Can anyone explain the difference between git add and git commit?

On-stage Task: Ask a learner to demonstrate creating a file and committing changes.

3. .gitignore and Ignoring Files

Introduction to .gitignore: Discuss its purpose in preventing sensitive or unnecessary files from being tracked, e.g., .env files, logs, or dependencies (node_modules/).

Activity: Create a .gitignore file and add rules.

Engaging Questions:

  • What kinds of files should never be shared in a repository?
  • Why might you ignore a node_modules/ directory?

On-stage Task: Guide a learner to create a .gitignore file and demonstrate its effect.

4. GitHub Introduction

Registration & Setup: Guide learners to sign up on GitHub (if needed) and create a new repository via the GitHub web interface.

Connecting Git and GitHub:

  • Add a remote: git remote add origin.
  • Push changes: git push -u origin main.

Engaging Questions:

  • How does GitHub simplify collaboration?

On-stage Task: Invite a learner to create a branch, make changes, and resolve a conflict.

5. Collaborative Workflow

Branching: Create and switch branches using git branch and git checkout. Simulate feature development by modifying a file.

Merging: Merge changes using git merge and resolve conflicts (intentional conflict setup).

Engaging Questions:

  • Why is branching important in team projects?
  • What strategies would you use to avoid conflicts during merging?
  • How can branching help teams work on multiple features?
  • What steps can you take to avoid frequent merge conflicts?

On-stage Task: Invite a learner to create a branch, make changes, and resolve a conflict.

6. GitHub.dev Exploration

Introduction: Demonstrate GitHub.dev (press . in a GitHub repo).

Activity: Use GitHub.dev to edit the README.md file and commit changes directly.

Engaging Questions:

  • How might GitHub.dev help if your local environment isn't available?
  • What are the benefits of browser-based development?
  • What limitations do you see with GitHub.dev compared to local development?

On-stage Task: Have a learner open a repo in GitHub.dev and make an edit.
