Git & GitHub Practical Tutorial

Objective

Create a repository, make changes, and understand how to push these changes to GitHub. This exercise will help you gain a basic understanding of how to initialize a repository, make commits, and work with GitHub as a remote.

Prerequisites

  • Git installed on your computer
  • GitHub account created
  • Basic understanding of HTML and CSS
  • Text editor (VS Code recommended)

Detailed Steps

1. Set Up and Initialize Git

First, configure your Git identity. Open terminal/command prompt and run:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

2. Create a Local Repository

  1. Create a new folder for your project: 
    mkdir my-first-website
cd my-first-website
  2. Initialize the repository: 
    git init

3. Create and Add Files to the Repository

Create two files: index.html and styles.css

Content for index.html:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Git Project</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <div class="container">
        <h1>Hello, Git!</h1>
        <p>This is my first version-controlled website.</p>
    </div>
</body>
</html>

Content for styles.css:

.container {
    max-width: 800px;
    margin: 0 auto;
    padding: 20px;
    text-align: center;
}

h1 {
    color: #2c3e50;
}

p {
    color: #34495e;
}

4. Commit Changes

  1. Check the status of your repository: 
    git status
  2. Add the files to the staging area: 
    git add index.html styles.css
  3. Commit the changes: 
    git commit -m "Initial commit: Add basic website files"

5. Push Changes to GitHub

  1. Go to GitHub and create a new repository (without initializing with README)
  2. Connect your local repository to GitHub: 
    git remote add origin https://github.com/your-username/my-first-website.git
  3. Push your changes to GitHub: 
    git push -u origin main
    Note: If you're using an older version of Git, you might need to use master instead of main

6. Verify and View Changes on GitHub

  1. Go to your GitHub repository URL
  2. You should see your files listed in the repository
  3. Click on the files to view their contents
  4. Check the commit history by clicking on "commits"

Common Issues and Solutions

  • If you get authentication errors, make sure you're logged into GitHub
  • If git push fails, check if your repository URL is correct
  • If files aren't showing up in staging, verify that you're in the correct directory
